# simulations/

Stores STAR-CCM+ simulation files. All `.sim` files are gitignored.

## Expected files

- **Template `.sim`** — your starting geometry/physics/mesh template. Must already contain regions `fluid`, boundaries `inlet`, `jet_top`, `jet_bot`, and all mesh controls expected by the macros.
- **`cylinder_afc_Re100.sim`** — generated by `template_base_for_cylinderAFC.java` after applying `config/case_config.json`. This is the file used by `add_realtime_jets_from_json.java` at the start of each DRL episode.

## Generating the baseline sim

### GUI mode

Open your template `.sim` in STAR-CCM+, set the env var, then run the macro from Tools → Macros:

```powershell
$env:STARCCM_JSON = (Resolve-Path "config/case_config.json").Path
# then: Tools → Macros → Run Macro → macros/template_base_for_cylinderAFC.java
```

### Batch mode

```powershell
$env:STARCCM_JSON = (Resolve-Path "config/case_config.json").Path
& "C:\...\starccm+.exe" -batch -macro macros/template_base_for_cylinderAFC.java simulations/your_template.sim
```

The macro meshes the geometry, initializes the flow with jets off, runs `time.steps` timesteps to develop vortex shedding, then saves `cylinder_afc_Re100.sim` to this folder.

After saving, run `add_history_points.java` on the saved sim to create probe monitors before DRL training.
